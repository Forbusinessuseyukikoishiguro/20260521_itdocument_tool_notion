## 🔁 23モードを貫く「同じ型」

このツールの本質は、**1つのパターンの繰り返し**です。どのモードも：

```
1. data-○mode 属性つきカードをクリック → モード変数に記録
2. data-○mode-target が一致するフォームだけ表示
3. 生成ボタン click
4. v('id') で入力を集める
5. テンプレートリテラルでMarkdown文字列を組み立てる
   （空欄は ${x || 'デフォルト'} で埋める）
6. textContent で出力エリアに入れる
7. copyTo / downloadMarkdown で外に出す
```

この7ステップさえ頭に入れれば、8,250行のどこを開いても「ああ、ここはステップ5の組み立て部分だな」と読めます。

新しいモードを足すときも同じ。

```
① HTMLに選択カード（data-○mode）とフォームブロック（data-○mode-target）を追加
② JSのMETA定数にモード名を登録
③ 生成の分岐（else if）に組み立て処理を追加
```

この3点セットを守れば壊れません。実際 v2.5 までの全機能拡張は、すべてこの型の繰り返しで作られています。

最後に、なぜこの巨大な1ファイルが壊れずに動くのか。リリース前に毎回これを通しているからです。

